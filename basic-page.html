# 4. ページセレクタとページのコンテクスト ## 4.1. ページルール ページボックスの寸法、向き、余白など、さまざまな側面を@pageルールで指定できます。ページ・ルールは、ルール・セットが使用できる場所であればどこでも使用できます。ページ・ルールは、キーワード @page、コンマで区切られたページ・セレクタのリスト（オプション）、宣言のブロック（ページ・コンテキストと呼ばれる）で構成されます。また、@pageルールには、宣言の間に他のアットルールを挿入することができます。現在の仕様では、@page内のmargin at-rulesのみが許可されています。 セレクタ・リストのない@pageルールはすべてのページに適用されます。その他の@page規則は、少なくとも1つのセレクタにマッチするページに適用されます。ページ・コンテキスト内で宣言されたプロパティは、ページ・ボックスに適用されます。 ページまたはマージン・コンテキスト内の宣言ブロックの処理中にエラーが発生した場合は、「解析エラーの処理に関する規則」が適用されます。 # 4. ページセレクタとページのコンテクスト ## 4.1. ページルール ページボックスの寸法、向き、余白など、さまざまな側面を@pageルールで指定できます。ページ・ルールは、ルール・セットが使用できる場所であればどこでも使用できます。ページ・ルールは、キーワード @page、コンマで区切られたページ・セレクタのリスト（オプション）、宣言のブロック（ページ・コンテキストと呼ばれる）で構成されます。また、@pageルールには、宣言の間に他のアットルールを挿入することができます。現在の仕様では、@page内のmargin at-rulesのみが許可されています。 セレクタ・リストのない@pageルールはすべてのページに適用されます。その他の@page規則は、少なくとも1つのセレクタにマッチするページに適用されます。ページ・コンテキスト内で宣言されたプロパティは、ページ・ボックスに適用されます。 ページまたはマージン・コンテキスト内の宣言ブロックの処理中にエラーが発生した場合は、「解析エラーの処理に関する規則」が適用されます。 # 4. ページセレクタとページのコンテクスト ## 4.1. ページルール ページボックスの寸法、向き、余白など、さまざまな側面を@pageルールで指定できます。ページ・ルールは、ルール・セットが使用できる場所であればどこでも使用できます。ページ・ルールは、キーワード @page、コンマで区切られたページ・セレクタのリスト（オプション）、宣言のブロック（ページ・コンテキストと呼ばれる）で構成されます。また、@pageルールには、宣言の間に他のアットルールを挿入することができます。現在の仕様では、@page内のmargin at-rulesのみが許可されています。 セレクタ・リストのない@pageルールはすべてのページに適用されます。その他の@page規則は、少なくとも1つのセレクタにマッチするページに適用されます。ページ・コンテキスト内で宣言されたプロパティは、ページ・ボックスに適用されます。 ページまたはマージン・コンテキスト内の宣言ブロックの処理中にエラーが発生した場合は、「解析エラーの処理に関する規則」が適用されます。 # 4. ページセレクタとページのコンテクスト ## 4.1. ページルール ページボックスの寸法、向き、余白など、さまざまな側面を@pageルールで指定できます。ページ・ルールは、ルール・セットが使用できる場所であればどこでも使用できます。ページ・ルールは、キーワード @page、コンマで区切られたページ・セレクタのリスト（オプション）、宣言のブロック（ページ・コンテキストと呼ばれる）で構成されます。また、@pageルールには、宣言の間に他のアットルールを挿入することができます。現在の仕様では、@page内のmargin at-rulesのみが許可されています。 セレクタ・リストのない@pageルールはすべてのページに適用されます。その他の@page規則は、少なくとも1つのセレクタにマッチするページに適用されます。ページ・コンテキスト内で宣言されたプロパティは、ページ・ボックスに適用されます。 ページまたはマージン・コンテキスト内の宣言ブロックの処理中にエラーが発生した場合は、「解析エラーの処理に関する規則」が適用されます。
